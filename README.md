# 🗄️ Projet Système de Gestion de Base de Données (SGBD)

Ce dépôt contient l'intégralité d'un projet de conception et d'implémentation d'une base de données relationnelle, réalisé dans le cadre de notre cursus informatique. Le projet porte sur la gestion d'une **Boutique**.

## 🚀 Structure du Projet

Le projet est divisé en scripts SQL numérotés pour respecter l'ordre logique de déploiement d'une base de données :

1.  **`1_creation.sql`** : Script de création des tables (DDL).
2.  **`2_contraintes.sql`** : Définition des clés étrangères, contraintes d'intégrité et de vérification.
3.  **`3_insertion.sql`** : Jeu de données de test pour remplir la base.
4.  **`4_interrogation.sql`** : Requêtes complexes (SELECT, JOIN, etc.) pour extraire des informations pertinentes.

## 📊 Modélisation (MCD / MLD)
- Le fichier **`MCD_Boutique.loo`** contient le Modèle Conceptuel de Données.
- La conception a été assistée par des prompts structurés (présents dans les fichiers `.docx`) pour optimiser la structure relationnelle.

## 🛠️ Fonctionnalités couvertes
- Création d'une structure relationnelle cohérente.
- Gestion des relations entre entités (Clients, Produits, Commandes, etc.).
- Mise en place de contraintes métier.
- Analyse de données via des requêtes SQL avancées.

## 📥 Comment utiliser ces scripts
Pour tester la base de données : Utilisez un environnement SQL (comme MySQL)
